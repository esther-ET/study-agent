## 什么是智能体（Agent）？

**智能体（Agent）** = 一个能自主决策、调用工具、处理复杂任务的程序。

类比：
- **普通程序**：你写好代码，程序按固定流程执行
- **智能体**：你描述目标，智能体自主决定怎么做

### 智能体的核心能力

| 能力 | 说明 | 例子 |
|------|------|------|
| **感知** | 理解用户输入 | 解析"找2020年以后的目标检测论文" |
| **推理** | 分析意图，决定行动 | 提取年份、排序方式等参数 |
| **行动** | 调用工具完成任务 | 调用搜索API、生成摘要 |
| **记忆** | 保持上下文状态 | 记住已选择的论文 |

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户输入                                  │
│              "point cloud object detection"                      │
└─────────────────────┬───────────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Intent Parser（意图解析）                      │
│  - 解析用户查询                                                   │

                      │
                      ▼

│                  Search Executor（搜索执行）                      │
│  - 调用 UnifiedSearchClient 搜索论文                              │
│  - 多路搜索 + 相关性排序                                          │
│  输出: list[Paper]                                               │
└─────────────────────┬───────────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Formatter（格式化）                           │
│  - 将论文列表格式化为 Markdown                                     │
│  输出: "### 1. VoxelNet\n**年份**: 2018..."                      │
└─────────────────────┬───────────────────────────────────────────┘

                      ▼
┌─────────────────────────────────────────────────────────────────┐
│                       CLI 输出                                   │
└─────────────────────────────────────────────────────────────────┘
```

**这就是 LangGraph 的核心理念**：用**图（Graph）** 表示**工作流（Workflow）**，用**状态（State）** 在节点间传递数据。

## 核心概念：State（状态）

### 什么是 State？

**State** = 智能体在执行过程中需要记住的所有信息。

```python
# src/agent/state.py
class SearchState(BaseModel):
    user_input: str = ""           # 用户原始输入
    parsed_query: dict = {}        # 解析后的查询参数
    year_filter: int = 5           # 年份过滤（默认近5年）
    sort_by: str = "relevance"     # 排序方式
    papers: list[Paper] = []        # 搜索到的论文
    selected_papers: list[Paper] = []  # 用户选中的论文
    export_format: str = None       # 导出格式
    error: str = None               # 错误信息
```

### 为什么需要 State？

1. **共享数据**：多个节点需要访问相同的数据
2. **追踪进度**：记住执行到哪一步
3. **持久化**：用户的操作不会丢失（如选中的论文）

### State 的流动

```
初始State: {"user_input": "点云目标检测", "papers": [], ...}
    ↓
Intent Parser 修改: {"user_input": "点云目标检测", "parsed_query": {...}, ...}
    ↓
Search Executor 修改: {"user_input": "点云目标检测", "papers": [Paper1, Paper2, ...], ...}
    ↓
Formatter 修改: {"user_input": "...", "formatted_output": "### 1. ..."}
```

## 核心概念：Node（节点）

### 什么是 Node？

**Node** = 工作流中的一个处理步骤，接收输入（State），返回输出（State 的更新部分）。

### Node 的特点

1. **单一职责**：每个节点只做一件事
2. **无副作用**：给定相同输入，总是产生相同输出
3. **可复用**：节点可以被多个工作流复用

### Node 示例

```python
# src/agent/nodes/intent_parser.py
def parse_intent(state: SearchState) -> dict:
    """
    节点函数签名：
    - 输入：完整的 SearchState
    - 输出：需要更新的字段（dict形式）

    返回的dict会与原state合并，形成新的state
    """
    user_input = state.user_input

    # 解析年份
    year_from = None
    if "2020" in user_input:
        year_from = 2020

    # 返回需要更新的字段
    return {
        "parsed_query": {
            "query": user_input,
            "year_from": year_from,
            "sort_by": "relevance"
        }
    }
```

## 核心概念：Graph（图）

### 什么是 Graph？

**Graph** = 由 Node 和 Edge（边）组成的有向图，定义工作流的执行顺序。

```python
# src/agent/graph.py
from langgraph.graph import StateGraph, END

def build_graph():
    workflow = StateGraph(SearchState)  # 指定状态类型

    # 1. 添加节点
    workflow.add_node("intent_parser", parse_intent)
    workflow.add_node("search_executor", execute_search)
    workflow.add_node("formatter", format_node)

    # 2. 设置入口
    workflow.set_entry_point("intent_parser")

    # 3. 添加边（执行顺序）
    workflow.add_edge("intent_parser", "search_executor")  # intent_parser 完成后执行 search_executor
    workflow.add_edge("search_executor", "formatter")
    workflow.add_edge("formatter", END)  # 结束

    # 4. 编译
    return workflow.compile()

# 全局实例
app = build_graph()
```

### 图的可视化

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────┐
│ intent_parser │ ──▶ │ search_executor  │ ──▶ │  formatter   │
└──────────────┘     └──────────────────┘     └──────────────┘
                                                 │
                                                 ▼
                                               END
```

## 数据流

### 执行过程

```python
# 调用智能体
initial_state = SearchState(user_input="点云目标检测")
result = app.invoke(initial_state)
```

### invoke 内部发生了什么？

```
1. 创建初始状态
   state = {"user_input": "点云目标检测", "parsed_query": {}, ...}

2. 进入 intent_parser 节点
   new_fields = parse_intent(state)
   state.update(new_fields)
   state = {"user_input": "点云目标检测", "parsed_query": {"query": "...", ...}, ...}

3. 进入 search_executor 节点
   new_fields = execute_search(state)
   state.update(new_fields)
   state = {"papers": [Paper1, Paper2, ...], ...}

4. 进入 formatter 节点
   new_fields = format_node(state)
   state.update(new_fields)
   state = {"formatted_output": "### 1. ...", ...}

5. 返回最终状态
```

### State 的就地修改

LangGraph 的一个关键特性是：**节点返回的 dict 会与当前状态合并**，而不是替换整个状态。

```python
# 节点返回
return {"papers": [Paper1, Paper2]}  # 只更新 papers 字段

# 其他字段保持不变
# state.parsed_query, state.year_filter 等仍然保留
```

## 代码解读

### 1. 搜索执行器（带多路搜索优化）

```python
# src/tools/openalex_client.py
class OpenAlexClient:
    def search(self, query, year_filter=None, limit=10, use_multi_search=True):
        """
        搜索论文

        关键优化：长查询使用多路搜索
        - 将长查询分词为多个子查询
        - 并行搜索后按匹配次数评分
        - 解决 OpenAlex 对长查询相关性退化的问题
        """
        if use_multi_search and len(query.split()) >= 3:
            return self._multi_search(query, year_filter, limit)
        return self._search_single(query, year_filter, limit)

    def _multi_search(self, query, year_filter, limit):
        """多路搜索：分词 -> 并行搜索 -> 合并去重 -> 按匹配分数排序"""
        # 1. 分词生成子查询
        words = query.lower().split()
        sub_queries = [query]  # 保留完整查询
        for i in range(len(words) - 2):
            sub_queries.append(' '.join(words[i:i+3]))  # 3词组合

        # 2. 并行搜索
        paper_scores = {}
        with ThreadPoolExecutor(max_workers=4) as executor:
            futures = {executor.submit(self._search_single, q, year_filter, limit): q
                      for q in sub_queries[:5]}
            for future in as_completed(futures):
                papers = future.result()
                for paper in papers:
                    # 记录匹配次数
                    if paper.paper_id not in paper_scores:
                        paper_scores[paper.paper_id] = (paper, 0)
                    paper_scores[paper.paper_id] = (paper, paper_scores[paper.paper_id][1] + 1)

        # 3. 按匹配分数排序
        results = [p for p, _ in sorted(paper_scores.values(), key=lambda x: x[1], reverse=True)]
        return results[:limit]
```

### 2. LLM 客户端（统一接口）

```python
# src/tools/llm_client.py
def call_llm(prompt, system_prompt=None, provider=None, model=None):
    """
    统一LLM调用接口

    设计模式：工厂模式 + 配置驱动
    - 通过配置切换provider（minimax/deepseek）
    - 通过配置切换模型
    """
    p = provider or get_current_provider()  # 从配置读取
    api_key = get_api_key(p)
    model_name = model or get_model_name(p)

    # 统一的API调用
    response = requests.post(
        API_ENDPOINTS[p],
        headers={"Authorization": f"Bearer {api_key}"},
        json={"model": model_name, "messages": [...]}
    )
    return response.json()["choices"][0]["message"]["content"]
```

### 3. 导出功能

```python
# src/utils/export.py
def paper_to_bibtex(paper):
    """论文转 BibTeX 格式"""
    first_author = paper.authors[0].split()[-1] if paper.authors else "unknown"
    cite_key = f"{first_author}{paper.year}"  # 生成引用键

    # 根据会议名称判断类型
    entry_type = "inproceedings" if paper.venue in ["CVPR", "ICCV", "NeurIPS"] else "article"

    return f"""@{entry_type}{{{cite_key},
  title = {{{paper.title}}},
  author = {{{' and '.join(paper.authors)}}},
  year = {{{paper.year}}},
}}"""
```

## 关键设计模式

### 1. 配置驱动

```python
# configs/settings.py
class Settings(BaseSettings):
    llm_provider: str = "minimax"  # 从环境变量读取
    minimax_api_key: str = ""
    deepseek_api_key: str = ""
```

**好处**：不修改代码，只改配置就能切换行为。

### 2. 单一职责

每个文件只负责一件事：

| 文件 | 职责 |
|------|------|
| `state.py` | 定义数据结构 |
| `intent_parser.py` | 解析用户意图 |
| `search_executor.py` | 执行搜索 |
| `formatter.py` | 格式化输出 |
| `export.py` | 导出文件 |

### 3. 串行 vs 并行

```python
# 串行：顺序执行，总时间 = A + B + C
result_a = do_a()
result_b = do_b(result_a)
result_c = do_c(result_b)

# 并行：同时执行，总时间 = max(A, B, C)
with ThreadPoolExecutor(max_workers=4) as executor:
    future_a = executor.submit(do_a)
    future_b = executor.submit(do_b)
    results = [f.result() for f in as_completed([future_a, future_b])]
```

